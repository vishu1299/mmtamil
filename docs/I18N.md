# Internationalization (English / Tamil)

## How it works

- **Locales:** `en`, `ta` (see `lib/i18n/config.ts`).
- **Messages:** `messages/en.json` + `messages/en.app.json` (and `ta` equivalents) are **deep-merged** in `lib/i18n/load-messages.ts`.
- **Provider:** `app/_components/i18n/locale-provider.tsx` wraps the app in `app/providers.tsx`.
- **Preference:** `localStorage` key `preferred-language` and optional cookie `NEXT_LOCALE`; API field `profile.appLanguage` (`ENGLISH` / `TAMIL`).

## Adding / translating UI text

1. Add the key to **both** `messages/en.app.json` (or `en.json`) **and** `messages/ta.app.json` (or `ta.json`) under a namespace, e.g. `notifications.title`.
2. In a **client** component: `const t = useTranslations("notifications");` then `{t("title")}`.
3. For strings with variables: `{t("pendingCount", { count: 5 })}` and in JSON: `"pendingCount": "{count} pending"`.

## Still to migrate (same pattern)

Use `grep` for hardcoded English in:

- `app/(pages)/notifications/`, `chat/`, `premium/`, `credits/`, `payments/`, `checkout/`
- `app/(pages)/search/` (filter, carousel, details)
- `app/(pages)/settings/*` (edit-profile, password, deactivate, delete, notifications)
- `app/(auth)/loginstep/**` (all steps)
- Legal / privacy / support / about pages (large blocks of copy)
- `yourprofile.tsx`, mail components, news-feed, following, people, etc.

**Note:** User-generated or API text is not translated unless you map values or the API returns localized strings.
